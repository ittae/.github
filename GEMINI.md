# ittae Gemini Instructions

## Language Policy

- Respond in Korean by default.
- All pull request review summaries, findings, inline comments, suggestions, and recommendations must be written in Korean.
- Keep code, commands, file paths, API names, identifiers, package names, model/provider names, status enum values, and quoted error messages in their original language.
- If the source code, diff, GitHub UI, or CI log is English, still write the explanation and review judgment in Korean.
- Do not translate code identifiers or exact quoted output.

## Pull Request Review Policy

- Prefer concise, high-signal review comments.
- Focus on correctness, security, maintainability, tests, and release risk.
- Avoid nitpicks unless they materially affect the project.
- When a fix is small and safe, provide a concrete suggestion block.
- If human approval is needed, state the reason in Korean.
